This guide covers our specific workflows and custom features within Grafana.  
For general usage, refer to the [official Grafana documentation](https://grafana.com/docs/grafana/latest/).

## Saving Dashboards via Pull Request - WIP

Since we are running Grafana v12, you no longer need to manually copy JSON models to update dashboards in our repositories. You can save your changes directly from the UI.

1. Make your desired edits to a dashboard in the Grafana UI.
2. Click the **Save** button.
3. Select the option to open a **Pull Request** directly to the repository.
4. Grafana will automatically generate the PR with your JSON changes.

## Adding Grafana MCP to an LLM

You can leverage our custom LLM integration to build and modify dashboards using natural language.  
To do this, you need to configure your LLM client (like Gemini, Claude Desktop, etc.) to use the Grafana MCP server.

Here is an example of the settings you need to add to your agent's configuration file:
```json
{
  "mcpServers": {
    "grafana": {
      "command": "uvx",
      "args": ["mcp-grafana"],
      "env": {
        "GRAFANA_URL": "[https://grafana.mapcolonies.net](https://grafana.mapcolonies.net)",
        "GRAFANA_API_KEY": "GRAFANA_SERVICE_ACCOUNT_TOKEN"
      }
    }
  }
}
```

## Monitoring Redis

If you are deploying your own Redis instance, prefer using installed `redis-app` plugin instead of using redis-exporter

1. Navigate to the **Redis** app plugin in the Grafana sidebar.
2. Use the pre-built, ready-to-use dashboards provided by the plugin.
3. Utilize the plugin's custom UI to query your Redis instances directly without needing to write complex PromQL.
