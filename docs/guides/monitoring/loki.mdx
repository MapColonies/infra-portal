This guide covers how to search and analyze logs within our environment. For advanced LogQL (Loki Query Language) syntax, 
refer to the [official Grafana Loki documentation](https://grafana.com/docs/loki/latest/).

## Querying Logs in Grafana

Since Loki does not have its own standalone UI, all log querying is done through Grafana.

1. Open Grafana and navigate to the **Explore** tab (the compass icon in the left sidebar).
2. In the top-left data source dropdown, select **Loki**.
3. Use the **Label filters** (e.g., selecting `app` = `my-backend-service` or `namespace` = `production`) to narrow down your log streams.
4. Click **Run query** to view the logs.

## Jumping from Traces to Logs

One of the most powerful features of our observability stack is the seamless integration between **Tempo** (traces) and **Loki** (logs). 

If you are investigating a slow or failed request using a trace in Tempo, look for a small document icon next to the relevant span. 
Clicking this icon will automatically open a split pane in Grafana, querying Loki for the exact logs generated by that specific service during the exact timeframe of that span!
