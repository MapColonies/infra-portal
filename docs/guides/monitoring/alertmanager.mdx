This guide covers how developers interact with Alertmanager. 
For a deeper understanding of its core concepts, refer to the [official Alertmanager documentation](https://prometheus.io/docs/alerting/latest/alertmanager/).

## Viewing and Managing Active Alerts

While you cannot change the global routing rules, you have full access to the Alertmanager UI to view and manage active alerts.

1. Open [Alertmanager](https://alertmanager.mapcolonies.net).
2. The main dashboard will show all currently firing alerts, grouped by their labels.
3. If an alert is a known issue being worked on, you can click **Silence** to temporarily suppress notifications for that specific alert group. 
This helps reduce alert fatigue for the rest of the team.

![Alertmanager active alert](/img/monitoring/alertmanager/alertmanager-alert.png)

## Requesting Routing Changes

:::note Infrastructure Managed
The routing configuration (deciding which alerts go to which Slack channels or emails) is global and centrally managed by the **infra team**.
:::

Developers currently have **read-only access** to the existing routing configurations. You do not need to configure routing for your applications yourself. 

If your team needs a new Slack channel added, or if you want specific service alerts routed to a different destination, 
please contact the **infra team** to discuss adjusting the global configuration.
