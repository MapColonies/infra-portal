The Redis Exporter provides visibility into the performance, memory usage, and health of Redis databases across our environment.

:::tip Recommended Tool: Redis App Plugin
While the Redis Exporter collects the raw metrics, **we highly recommend using the `redis-app` Grafana plugin** for all your monitoring and visualization needs. 
The plugin provides comprehensive, ready-to-use dashboards and a custom UI for querying your databases directly. 

You should only rely on the raw metrics provided by this exporter as a fallback if the `redis-app` plugin does not expose a specific, edge-case metric you desire.
:::

## Central vs. Developer-Deployed Instances

We have two primary ways Redis is used and monitored in our stack:

1. **Managed Central Redis:** The infrastructure team maintains a central, managed Redis instance. 
The infra `redis-exporter` is configured exclusively to monitor this specific managed instance.
2. **Developer-Deployed Redis:** If your team deploys its own custom Redis instances alongside your applications, 
the exporter is already built directly into the standard deployment chart.
