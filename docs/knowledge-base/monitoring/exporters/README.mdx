Exporters are specialized, lightweight binaries or containers that act as adapters between third-party systems and Prometheus. Since not every application or service natively exposes metrics in the Prometheus format, exporters fetch data from these external systems, translate it, and serve it on a standard `/metrics` endpoint for Prometheus to scrape.

## Our Exporter Stack

We utilize a carefully selected set of exporters to provide deep visibility into our infrastructure, databases, and managed services without requiring you to write custom metric-gathering logic:

* **[Blackbox Exporter](./blackbox-exporter.mdx):** Actively probes external and internal HTTP/HTTPS endpoints to monitor uptime, response times, and general availability.
* **[Postgres Exporter](./postgres-exporter.mdx):** Gathers deep performance, connection, and health metrics from our central PostgreSQL database.
* **[Cert Exporter](./cert-exporter.mdx):** Monitors TLS certificate expiration dates across the cluster to prevent unexpected outages.
* **[Kube State Metrics (KSM)](./kube-state-metrics.mdx):** Listens directly to the Kubernetes API to generate metrics about the state of our deployments, pods, allocations, and nodes.
* **[Redis Exporter](./redis-exporter.mdx):** Exposes memory, connection, and performance metrics for both central and application-specific Redis instances.
