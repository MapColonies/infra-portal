Kube State Metrics (KSM) is a service that listens to the Kubernetes API server and generates metrics about the state of the objects inside our cluster. While the Blackbox Exporter probes from the outside, KSM tells us exactly what is happening under the hood with our Deployments, Pods, Nodes, and other standard Kubernetes resources.

## What It Monitors

KSM is configured to track the default, standard Kubernetes resources out of the box. We do not currently use it to expose metrics for Custom Resource Definitions (CRDs).

Because it tracks the cluster state globally, **zero developer configuration is required**. You do not need to add annotations or configure your deployments to be monitored by KSM.

## Common Use Cases

KSM is incredibly useful for troubleshooting and optimizing your applications. Here are two primary ways developers leverage KSM data in Grafana:

* **Tracking Crash Loops:** You can build dashboards or set alerts to detect when your Pods enter a `CrashLoopBackOff` state, helping you catch startup failures or recurring application crashes immediately.
* **Resource Allocation vs. Actual Usage:** KSM exposes the CPU and memory limits/requests you have set for your Pods. By comparing these KSM allocation metrics against your actual usage metrics, you can easily spot if your services are over-provisioned (wasting resources) or under-provisioned (risking performance bottlenecks).
