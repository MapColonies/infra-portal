# Monitoring Stack Overview

Welcome to the monitoring stack! This Kubernetes-based setup gives us full visibility into our cluster's health, performance, and application behavior. 

Our stack leverages a powerful suite of open-source tools to handle metrics, logs, and traces seamlessly, providing a unified observability experience.

## Architecture Overview

Here is a high-level look at how data flows through our system:

* **Metrics:** Application Pods and specialized Exporters expose metrics. **Prometheus** scrapes these metrics, 
feeding the data to **Grafana** for visualization and triggering alerts via **Alertmanager**.
* **Traces:** Applications send traces to the **OpenTelemetry Collector** (`infra-otel`), 
which smartly routes and forwards them to **Tempo** (backed by S3) for querying in Grafana.
* **Logs:** Application logs are picked up by **Grafana Alloy** (via pod labels or direct push) and sent to **Loki** (backed by S3) for searching and analysis in Grafana.

## General Guidelines & Features

* **Save Dashboards via PR:** We are running Grafana v12, which allows you to save your dashboard modifications directly by opening a Pull Request 
from within the Grafana UI!  
(We are also working on fully automated dashboard provisioning via sidecar ConfigMaps [soon](https://mapcolonies.atlassian.net/browse/MAPCO-10563)).
* **AI-Powered Dashboards (Grace Period):** We've integrated an LLM into Grafana via MCP using a custom service account! You can use it to auto-generate and modify dashboards.  
**Important:** During our grace period, the AI is restricted to creating and modifying dashboards *only* inside the `AI` folder to keep our core dashboards safe.
* **Alert Routing:** Alertmanager handles routing notifications to Slack, classified chats, and email. This global configuration is centrally managed by the **infra team**. 
If you need a new route, please reach out.  
*(Heads up: We will be migrating from Alertmanager to Grafana Alerts in the near future!)*
